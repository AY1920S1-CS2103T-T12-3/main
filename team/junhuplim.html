1. Introduction

The purpose of this portfolio showcases my contribution to the Athletick project.

1.1. About the team

We are a team of five year 2 computing students from National University of Singapore with diverse backgrounds and specialisations.

1.2. About the project

This project is a 6 weeks long project for our software engineering module. We were given an existing address book application and were tasked to either enhance or to morph it. One of the main constraints of this project is that the application has to be used as a command line interface.

What my team and I decided to do was to morph the existing address book application to a team management desktop application called Athletick. Athletick is designed for captains and coaches of timing-based performance sports to help take and keep track of attendance and performance of their teams. Some of the main features of Athletick include taking the team’s attendance, viewing the team’s attendance rate and also tracking the team’s performance with a visual graph.

2. Summary of contributions

This section shows a summary of my coding, documentation, and other helpful contributions to the team project.

Enhancement added: I added the ability to undo/redo previous commands.

  • What it does: The undo command allows the user to undo a previous command. The user may reverse this undo command with the redo command.

  • Justification: In the event that users have made a mistake or changed their minds about executing a command, the undo command enables them to revert to a version immediately before the mistaken command was executed. If they change their minds again and decide to execute the command after all, then the redo command enables them to do so easily.

  • Highlights

    • This enhancement works with existing as well as future commands.

    • As there were a few ways to implement this enhancement, an in-depth analysis of design alternatives was necessary to weigh the different alternatives and decide on the implementation for this enhancement.

    • The implementation too was challenging as it required a full understanding of the major components of the application and the existing features implementations. Also, as the enhancement is able to work with future commands, the implementation of this enhancement has to be considerable of the future features implementations and allow the future enhancement for such features to be easily integrated into the current implementation.
